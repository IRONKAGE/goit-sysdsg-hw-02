# **Завдання 2: Інтеграція й оптимізація комунікації в складній розподіленій системі**

## **Опис завдання:**

Ви є головним архітектором системи для великої онлайн-платформи бронювання авіаквитків, яка обслуговує мільйони користувачів щомісяця. Система складається з кількох мікросервісів: сервісу бронювання, платіжного сервісу, сервісу сповіщень, сервісу аналітики, сервісу керування інвентарем і нового сервісу рекомендацій.

## **Вимоги до виконання:**

1. (*Опціонально*) Використовуйте діаграми для ілюстрації архітектури вашої системи.
2. Поясніть ваші рішення у вигляді детального звіту.
3. Включіть обговорення потенційних вузьких місць і можливих шляхів їх усунення.
4. Зверніть увагу на такі неочевидні аспекти, як вплив вибраних технологій на інші частини системи та можливість їх заміни в майбутньому.

## **Завдання:**

1. **Проектування комунікації між мікросервісами:**
   - **Завдання:** Визначте типи комунікації (синхронна, асинхронна) між кожною парою мікросервісів. Виберіть відповідні технології (**REST**, **gRPC**, **брокери повідомлень**, **GraphQL**) для реалізації цієї комунікації
   - **Деталі:** Обґрунтуйте свій вибір з точки зору продуктивності, надійності, вимог до затримки та зворотної сумісності. Врахуйте різні сценарії, зокрема пікові навантаження під час флеш-розпродажів
2. **Оптимізація продуктивності та масштабованості:**
   - **Завдання:** Спроектуйте архітектуру для платіжного сервісу, яка здатна обробляти такі пікові навантаження, як флеш-розпродажі та сезонні сплески активності
   - **Деталі:** Використайте такі механізми, як черги повідомлень, кешування, обмеження швидкості (rate limiting) та автоматичне масштабування. Розгляньте можливість використання брокерів повідомлень для асинхронної обробки запитів.
3. **Розробка стратегії обробки помилок:**
   - **Завдання:** Описати стратегію обробки помилок для критичних сценаріїв, наприклад невдачі в обробці платежу або недоступності сервісу інвентаризації.
   - **Деталі:** Включіть механізми повторних спроб, резервні стратегії та сповіщення про помилки. Як ви забезпечите, що користувач отримає відповідь навіть у разі збою одного з сервісів?
4. **Еволюція інтерфейсів і версіювання:**
   - **Завдання:** Запропонуйте стратегію версіювання для *API* сервісу бронювання, щоб забезпечити зворотну сумісність із поточними клієнтами під час внесення змін.
   - **Деталі:** Врахуйте можливість додавання нових функцій та підтримки старих версій *API*. Обговоріть, як *GraphQL* може допомогти в розв’язанні проблем з версіюванням.
5. **Інтеграція сервісу рекомендацій:**
   - **Завдання:** Спроектуйте інтеграцію нового сервісу рекомендацій, який використовує машинне навчання для персоналізації пропозицій авіаквитків.
   - **Деталі:** Виберіть технології для комунікації з іншими сервісами. Розгляньте використання подійно-орієнтованої архітектури для обробки даних у реальному часі.
